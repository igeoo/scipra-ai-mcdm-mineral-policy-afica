Supplementary Executive Summary
This SI provides three appendices that together form the formal, technical, and empirical foundation of SCIPRA. Appendix A proves that PCI and RPCI are bounded in [0,1] and that RPCI is strictly penalised by cross-domain dispersion. Appendix B provides the complete reproducible specification of the NLP-SVM pipeline. Appendix C derives every P, L, U score assigned to the five Marikana stakeholder groups with each value traceable to a primary source, a formal equation, and a calibrated uplift magnitude.
These appendices are designed for independent or combined reading. A reviewer focused on mathematical validity needs only Appendix A (proofs). A reviewer focused on AI/NLP methodology needs only Appendix B (pipeline). A reviewer focused on stakeholder scoring transparency needs only Appendix C (derivations). Audience notes at the start of each appendix direct reviewers to the relevant sections.
Reproducibility in one sentence: Every numerical result in the main text can be independently reproduced using (a) the equations and parameter values in Table SI-1; (b) the algorithms in Section B.2 with the settings in Table B5.2; and (c) the derivation steps in Sections C.3–C.7 using publicly accessible sources listed in Section B.4.3.
RPCI note (0.427 vs 0.407): The main text reports RPCI_raw = 0.427 using the un-normalized form (λ=0.10 makes D=1.05, a 5% difference). This appendix uses the definitive computational form RPCI = max (0, PCI₀−λσ(X))/(1+λ/2). Full reconciliation in Section A.6.

A.1.2 Feasible Domain and Key Definitions
X  =  { X ∈ [0,1]^n  :  wᵢ ≥ 0,  ∑ᵢ wᵢ = 1 }   (A.1)

Definitions under Assumptions A.1–A.4:
PCI₀ = ∑ᵢ wᵢXᵢ;    μ = PCI₀;    σ(X) = √(∑ᵢ wᵢ(Xᵢ−μ)²)   (Eqs. 4, 7, 6)
RPCI_raw = PCI₀ − λσ(X);    RPCI = max(0, PCI₀ − λσ(X)) / (1+λ/2)   (Eqs. 5; A.2)

Clarification on Equations (8) and (A.2): Equation (8) in the main text and Equation (A.2) here are identical: RPCI = max(0, PCI₀−λσ(X))/(1+λ/2). Both documents use this as the definitive computational form. Equation (A.2) is referenced throughout this appendix for self-contained reading; all numerical results in both documents are derived from this expression.
A.1.3 Degeneracy Region
The non-negativity floor in Equation (A.2) binds when PCI₀ ≤ λσ(X), defining the degeneracy region:
D₀  =  { X ∈ X :  PCI₀(X)  ≤  λσ(X) }   (A.3)
When X ∈ D₀, RPCI = 0 (complete policy divergence). For the Marikana calibration (λ=0.10), PCI₀ ∈ [0.456, 0.781] and λσ(X) ∈ [0.002, 0.029], so D₀ is never active in the empirical application.

A.2 Lemma A.1.1: Bounding Proof for σ(X) ≤ 0.5  (Main Text ‘Appendix A.1’)
Applies Assumptions A.1–A.2. Proof holds for any n ≥ 2.

Intuition: In any governance system where domain scores are bounded between 0 (complete failure) and 1 (perfect performance), the most extreme cross-domain divergence occurs when half the weight sits on 0 and half on 1 — like one domain performing perfectly while another collapses. The resulting standard deviation is exactly 0.5. This bound means the RPCI variance penalty λσ(X) can never exceed λ×0.5, giving the normalization divisor D=1+λ/2 its precise derivation.


Proof.

Step 1: Reduce to the two-point case.
By the Hoeffding (1963) variance bound, the variance of Xᵢ ∈ [0,1] with fixed mean μ is maximised when mass is concentrated at {0,1}. Consider Xᵢ ∈ {0,1} for all i.

Step 2: Parametrise.
Let p = ∑ᵢ:Xᵢ=1 wᵢ ∈ [0,1]. Then μ = p (Equation A.4).

Step 3: Compute variance.
Var(X)  =  p(1−p)² + (1−p)p²  =  p(1−p).   (A.5)

Step 4: Maximise over p ∈ [0,1].
f(p) = p(1−p) is concave; f′(p) = 1−2p = 0 gives p* = 1/2; f′′ = −2 < 0 confirms maximum. Therefore:
max Var(X)  =  (1/2)(1/2)  =  1/4.   (A.6)

Step 5: Conclude.
σ(X) = √Var(X) ≤ √(1/4) = 0.5.    Tightness at p = 1/2. ■   (A.7)


A.3 Theorem A.1: PCI and RPCI Boundedness (Proposition 1)
Applies Assumptions A.1–A.3. Uses Lemma A.1.1 and Eq. A.2.

Intuition: PCI₀ is a weighted average of scores that are each between 0 and 1, so the average must also lie between 0 and 1. RPCI inherits this bound because its numerator cannot exceed PCI₀ (the penalty only reduces it) and the divisor is always ≥1. The floor max(0,·) prevents the penalty from pushing RPCI below zero.


Proof (a): PCI₀ ∈ [0,1].
PCI₀ = ∑wᵢXᵢ ≥ ∑wᵢ×0 = 0   (lower);    PCI₀ ≤ ∑wᵢ×1 = 1   (upper). ■

Proof (b): RPCI ∈ [0,1].
Lower: max(0,·) ≥ 0; denominator (1+λ/2) > 0. Therefore RPCI ≥ 0.
Upper: numerator ≤ PCI₀ ≤ 1; denominator ≥ 1. When λ=0: RPCI = PCI₀/1 ≤ 1. When λ>0: RPCI < 1. Tightness: RPCI=0 at D₀; RPCI=1 when λ=0 and PCI₀=1. 
A.4  Full Derivation of RPCI Normalization (Main Text ‘Appendix A.2’, Equation 8)
Uses RPCI_raw range [−λ/2, 1] from Theorem A.1 and Lemma A.1.1.

The un-normalized RPCI_raw ∈ [−λ/2, 1]. A normalization is required that: (i) maps RPCI_raw to [0,1]; (ii) preserves strict monotone ordering (no reranking); (iii) reduces to PCI₀ when λ = 0.

Step 1: Establish range.
RPCI_raw ∈ [−λ/2, 1] (upper from PCI₀ ≤ 1; lower from PCI₀ ≥ 0 and σ ≤ 0.5).

Step 2: Apply non-negativity floor.
RPCI_floor = max(0, PCI₀ − λσ(X)) ∈ [0,1].   (A.8)

Step 3: Identify compression factor.
Original range width: 1−(−λ/2) = 1+λ/2. Denote as the normalization divisor:
D  =  1 + λ/2.   (A.9)

Step 4: Derive normalized form.
RPCI  =  max(0, PCI₀ − λσ(X))  /  (1 + λ/2).   (A.10)
Division by D preserves relative ordering (monotonicity) and maps the effective range to [0,1].

Step 5: Verify output range.
Lower: max(0,·) ≥ 0; D > 0 ⇒ RPCI ≥ 0. Upper: numerator ≤ 1; D ≥ 1 ⇒ RPCI ≤ 1. 
Summary: D = 1+λ/2 is the exact span of RPCI_raw from its minimum to its maximum. It is the unique linear divisor satisfying both RPCI ∈ [0,1] and strict monotone ordering (Remark A.4.1).



A.5 Theorem A.2: Strict Monotone Convergence (Proposition 2)
Requires λ > 0 (strict). Applies Assumptions A.1–A.4 and Eq. A.2.

Intuition: Holding the average domain score constant, a governance configuration where all domains are equal is always rated higher than one where the same average is achieved by one high-performing domain offsetting a failing one. The derivative ∂RPCI/∂σ = −λ/(1+λ/2) is the precise rate at which balance is rewarded: for λ=0.10 (Marikana calibration), each 0.10 increase in σ reduces RPCI by approximately 0.010 — a small but strictly non-zero governance quality discount.


Proof.

Part (a): Interior (PCI₀ > λσ(X)).
Floor not binding ⇒ RPCI(σ) = (μ − λσ) / (1+λ/2). Partial derivative extending Equation (10) of the main text:
∂RPCI/∂σ  =  −λ / (1+λ/2)  <  0   for all λ > 0.   (A.12)
Strictly affine decreasing in the interior.

Part (b): Boundary (X ∈ D₀).
RPCI(X) = 0. If X′ has lower σ and PCI₀ > λσ(X′), then RPCI(X′) > 0 and strict monotonicity holds. For Marikana (λ=0.10), D₀ is never active (PCI₀ ≥ 0.456 ≫ 0.10×0.290 = 0.029).

Part (c): Relationship to main text Eqs. (9)–(10).
Main text: ∂RPCI_raw/∂σ = −λ (un-normalized). Equation (A.12): ∂RPCI/∂σ = −λ/(1+λ/2) (normalized). Related by scale factor 1/D; both strictly negative for λ > 0.

B.1 Formal Mathematical Specification
B.1.1 TF-IDF Feature Weighting
tf-idf(t,d,D)  =  tf(t,d) × log[(1+|D|)/(1+df(t,D))]  +  1

where tf(t,d) = log(1+count(t,d)) (sublinear smoothing). TF-IDF was chosen over contextual embeddings (e.g. BERT) for three reasons: (i) interpretability — feature weights are directly inspectable alongside domain lexicons; (ii) proportionality — an 87-document specialised corpus does not justify transformer pre-training data requirements; (iii) transparency — TF-IDF features map directly to the lexicon scoring in Equation (B.1). Limitations relative to transformer models are acknowledged in Section B.4.
B.1.2 Salience Dimension Scoring
df_dim(d)  =  |{t ∈ tokens(d) : t ∈ V_dim|} / |{t ∈ tokens(d) : t ∈ V_all|}   (B.1)
U_adj(d)  =  clip( df_U(d) + 0.10×compound(d),  0,  1 )   (B.2)
The ±0.10 VADER ceiling was validated on a 20-document hold-out sample; exceeding 0.10 produced no improvement in annotator agreement.

B.1.3 Stakeholder Acceptance Score and PCI
acc(g)  =  (1/|D_g|) × ∑_{d∈D_g} pʲ(d)   (B.3)
S_score  =  ∑_{g∈G} SIC_g×acc(g)  /  ∑_{g∈G} SIC_g   (B.4)
Marikana SIC values: SIC_community=0.749, SIC_investor=0.770, SIC_labour=0.807, SIC_govt=0.703, SIC_NGO=0.686 (Table 3, main text). PCI = w₁I + w₂R + w₃S (Equation 1, main text; w₁=0.30, w₂=0.35, w₃=0.35).

B.4.1 Corpus Metadata and Dual-Use Caveat
The 87-document corpus spans 2010–2023. Mixed-source documents were attributed to the dominant stakeholder voice; n=4 equal-split documents received 0.5 weighting in acc(g) (Equation B.3) to avoid double-counting. Note: the same corpus serves dual functions — SVM training/validation and case argument evidence base. The S_score is therefore derived from documents that also inform the scenario characterisation (Section 4.3). This dual use is inherent to retrospective single-case validation and does not compromise internal consistency, but classification outputs should be interpreted as evidence of methodological coherence within the Marikana case, not as independent predictive validation. Out-of-sample validation across multiple cases remains future work.
