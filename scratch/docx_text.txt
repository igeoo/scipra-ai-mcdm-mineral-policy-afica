(1) Boundedness: both PCI₀ and RPCI lie in [0,1] for all domain scores in [0,1] and any convex weight vector (Theorem A.1). (2) Dispersion bound: the weighted standard deviation σ(X) ≤ 0.5 for all admissible score vectors, with the bound tight at maximum policy divergence (Lemma A.1.1). (3) Unique normalisation: the divisor D = 1 + λ/2 is the only linear scaling that simultaneously preserves RPCI ∈ [0,1] and the strict monotone ordering of Theorem A.2 (Remark A.4.1). (4) Monotone convergence: RPCI is strictly decreasing in cross-domain dispersion σ(X) for any λ > 0, rewarding policy balance and penalising sectoral optima (Theorem A.2 and Corollaries A.7.1–A.7.2). These guarantees hold for any n ≥ 2 domain scores, not only the three-domain SCIPRA case. A.1 Formal Setup: Assumptions, Feasible Domain, and Notation Key symbols in this section: Xᵢ = domain score for domain  i ; wᵢ = domain weight; μ = weighted mean; λ = dispersion-sensitivity parameter; σ(X) = weighted standard deviation; PCI₀ = base PCI; RPCI = robustness-adjusted PCI. A.1.1 Formal Assumptions (replacing main text informal list) The following four assumptions, informally stated in Section 3.3 of the main text, are here formalised as named assumptions referenced explicitly in all proofs. Assumption A.1 (Domain score bounds). Xᵢ  ∈  [0, 1] for all  i   ∈  {1,  … , n}, where n  ≥  2 is the number of policy domains. In the SCIPRA three-domain case,  i   ∈  {I, R, S} with n = 3. Assumption A.2 (Convex weight vector). Weights wᵢ ≥ 0 for all  i  and ∑ᵢ wᵢ = 1. The set {wᵢ} forms a convex combination. Assumption A.3 (non-negative dispersion sensitivity). λ ≥ 0. When λ = 0, RPCI reduces to PCI₀ (no dispersion penalty). When λ > 0, cross-domain dispersion is penalised. Assumption A.4 (Weighted mean). The weighted mean is μ = ∑ᵢ  wᵢX ᵢ. Note that μ = PCI ₀ by definition  (Equation 4, main text). A.1.2 Feasible Domain Space The feasible domain for all results in this appendix is: X  =    { X   ∈  [0, 1]^n  :   w ᵢ   ≥   0,   ∑ ᵢ  w ᵢ  =  1 }    (A.1) In the three-domain SCIPRA case, X = {(X_I, X_R, X_S) ∈ [0, 1]³  : w₁ + w₂ + w₃ = 1}. The domain X is a convex polytope (specifically, the product of the unit intervals weighted by a probability simplex). All suprema and infima referenced in the proofs are attained within X. A.1.4 Degeneracy Region The boundary condition at which the non-negativity floor in Equation (A.2) becomes binding is: PCI ₀  ≤   λ × σ(X)    ( Degeneracy condition) When this condition holds, RPCI = 0, indicating complete policy divergence — a governance state where the dispersion penalty fully offsets the mean policy score. For  SCIPRA’s  three empirical scenarios, PCI₀ ∈ [0.456, 0.781] and  λσ (X) ∈ [0.002, 0.029] (see Section A.6), so the degeneracy condition is never active in the Marikana application. The degeneracy region is formally defined as: D ₀  =    { X   ∈   X :   PCI ₀ (X )   ≤    λ   ×   σ (X ) }    (A.3) For the Marikana calibration (λ = 0.10), D₀ corresponds to extremely fragmented governance configurations with low mean performance and high cross-domain dispersion — pathological cases outside the empirical range of this study. A.2 Lemma A.1.1: Bounding Proof for σ(X) ≤  0.5  ( Main Text ‘Appendix A.1’) Notation: σ(X) = weighted standard deviation (Eq. 6); wᵢ ≥ 0, ∑wᵢ = 1 (Assumption A.2); Xᵢ  ∈  [0,1] (Assumption A.1). Proof holds for any n  ≥  2. Lemma A.1.1 (Upper Bound on Weighted Standard Deviation). Under Assumptions A.1 and A.2, for any n ≥ 2 and any convex weight vector {wᵢ} with ∑ᵢ wᵢ = 1: σ(X) ≤ 0.5 for all X  ∈  X. The bound is tight:  σ (X) = 0.5 is achieved when exactly half the total weight is on X ᵢ  = 0 and half on X ᵢ  = 1, representing maximum policy divergence. Proof. We maximise Var(X) = ∑ᵢ  wᵢ( Xᵢ −  μ)²  over X ∈ X, then take the square root. Step 1: Reduce to the two-point case. For a fixed mean μ, the variance is a convex functional of the distribution of Xᵢ. By the  Hoeffding  (1963) variance bound for bounded random variables, the variance over [0,1] with fixed mean μ is maximised by concentrating mass at the endpoints {0, 1}. We therefore consider the extreme case Xᵢ ∈ {0, 1} for all  i . Step 2: Parameterise the two-point distribution. Let p = ∑ ᵢ:Xᵢ =1 wᵢ, so (1−p) is the total weight on 0, and p ∈ [0,1] by Assumption A.2. The weighted mean is: μ  =   p × 1 + (1−p) ×  0  =   p.    (A.4) Step 3: Compute the two-point variance. Var(X )  =   p(1− p)²  + (1− p)p²  =   p(1−p).    (A.5) Step 4: Maximise over p  ∈  [0,1]. f(p) = p(1−p) is concave on [0,1]. Setting f′(p) = 1 − 2p = 0 gives p* = 1/2, confirmed as a maximum by f′′(p) = −2 < 0. Therefore: max Var(X )  =    f( 1/2 )  =   (1/ 2)( 1/2 )  =   1/4.    (A.6) Step 5: Conclude. σ(X )  =   √Var(X )  ≤    √( 1/4 )  =   0.5.    (A.7) Tightness: the bound is achieved exactly when p = 1/2, i.e., when total weight is equally split between Xᵢ = 0 and Xᵢ = 1 — the maximum policy divergence state. In the SCIPRA three-domain context, this corresponds to one or more domain scores at 0 and the others at 1 with equal combined weights: the governance state of complete incoherence. For the empirical Marikana scenarios, σ(X) ∈ [0.022, 0.290], confirming that all three scenarios lie well within the interior of the feasible dispersion range (see Section A.6). ■ ■ Remark A.2.1. Lemma A.1.1 holds for any n ≥ 2 policy domains, confirming that the RPCI framework generalises beyond the three-domain SCIPRA case to n-dimensional convex combinations. For n domains with equal weights (wᵢ = 1/n), the two-point maximum is attained when half the domains score 0 and half score 1, regardless of n. A.3 Theorem A.1: PCI and RPCI Boundedness (Proposition 1, Main Text) Notation: PCI₀ = base PCI (Eq. 4); RPCI = normalised robustness-adjusted PCI (Eq. A.2); λ ≥ 0 (Assumption A.3); σ(X) ≤ 0.5 (Lemma A.1.1). Theorem A.1 (Boundedness of PCI and RPCI — Proposition 1, main text). Under Assumptions A.1–A.3 and for any X  ∈  X: (a) PCI ₀   ∈  [0, 1]; and (b) RPCI  ∈  [0, 1]. Both bounds are tight. Proof of Part (a): PCI₀  ∈  [0, 1]. By Assumption A.1, Xᵢ ∈ [0,1]. By Assumption A.2, wᵢ ≥ 0 and ∑wᵢ = 1. Therefore: PCI ₀  =   ∑ᵢ  wᵢX ᵢ  ≥   ∑ᵢ wᵢ ×  0  =   0.    (lower bound) PCI ₀  =   ∑ᵢ  wᵢX ᵢ  ≤   ∑ᵢ wᵢ ×  1  =   1.    (upper bound) PCI₀ is a convex combination of values in [0,1] and is therefore contained in [0,1]. Tightness: PCI₀ = 0 when all Xᵢ = 0; PCI₀ = 1 when all Xᵢ = 1. ■ Proof of Part (b): RPCI  ∈  [0, 1]. From Equation (A.2): RPCI =  max( 0, PCI₀ −  λσ (X)) / (1 + λ/2). Lower bound:  The numerator  max( 0, ·) ≥ 0 and the denominator (1 + λ/2) > 0 by Assumption A.3. Therefore RPCI ≥ 0. Upper bound:  The numerator  max( 0, PCI₀ −  λσ (X)) ≤ PCI₀ ≤ 1. The denominator (1 + λ/2) ≥ 1. When λ = 0: RPCI = PCI₀ / 1 ≤ 1. When λ > 0: numerator < 1 and denominator > 1, so RPCI < 1. Hence RPCI ≤ 1 in all cases. Tightness: RPCI = 0 when X ∈ D₀ (degeneracy region, Equation A.3). RPCI = 1 when λ = 0 and PCI₀ = 1.  A.4 Full Derivation of RPCI Normalisation (Main Text ‘Appendix A.2’, Equation 8) Notation:  RPCI_raw  = PCI₀ −  λσ (X) (Eq. 5); σ(X) ≤ 0.5 (Lemma A.1.1); D = 1 + λ/2 = normalisation divisor. A.4.1 Problem Statement The un-normalised  RPCI_raw  = PCI₀ −  λσ (X) may be negative when the dispersion penalty exceeds PCI₀. The minimum value is  RPCI_raw  ≥ −λ/2 (from PCI₀ ≥ 0 and σ(X) ≤ 0.5). A normalisation procedure is required that: ( i ) maps  RPCI_raw  to [0,1]; (ii) preserves the strict monotone ordering of Theorem A.2 (no reranking of scores); and (iii) reduces to RPCI = PCI₀ when λ = 0. A.4.2 Five-Step Derivation Step 1: Establish the range of  RPCI_raw . Upper bound:  RPCI_raw  ≤ PCI₀ ≤ 1 (since  λσ (X) ≥ 0 and PCI₀ ≤ 1 from Theorem A.1a). Lower bound:  RPCI_raw  ≥ −λ/2 (from min PCI₀ = 0 and max σ(X) = 0.5, Lemma A.1.1). Therefore  RPCI_raw  ∈ [−λ/2, 1]. Step 2: Apply the non-negativity floor. RPCI_ floor   =    max(0,  PCI ₀ −  λσ (X ))   ∈   [0, 1].    (A.8) The floor eliminates the uninterpretable region [−λ/2, 0). The resulting range [0, 1] is correct, but the effective spread is compressed relative to the original range [−λ/2, 1]. Step 3: Identify the compression factor. The original range width before flooring is 1 − (−λ/2) = 1 + λ/2. Denote this as the normalisation divisor: D  =   1 + λ/2.    (A.9) Step 4: Derive the normalised form. Dividing  RPCI_floor  by D rescales the effective range to [0, 1/(1+λ/2)] ⊆ [0,1], which preserves the relative ordering of all scores (division by a positive constant is order-preserving): RPCI  =    max(0,  PCI ₀ −  λσ (X ))  /   (1 + λ/2).    (A.10) This is the definitive computational form (Equation A.2, this appendix). Equation (8) of the main text expresses this equivalently in shorthand multiplicative form; both produce identical values. See Section A.1.3 for the explicit clarification. Step 5: Verify output range. Lower:  max( 0, ·) ≥ 0; D > 0; therefore RPCI ≥ 0. Upper: max value of numerator = 1; D = 1 + λ/2 ≥ 1; therefore RPCI ≤ 1/D ≤ 1.  A.4.3 Summary of Derivation The normalisation divisor D = 1 + λ/2 is derived from the range width of  RPCI_raw : it is exactly the span from the minimum possible value (−λ/2) to the maximum (1). This makes D the unique linear scaling factor that maps  RPCI_raw  to a unit-bounded range while preserving strict monotone ordering — established in Remark A.4.1 below. Remark A.4.1 (Uniqueness of the Normalisation Divisor). The divisor D = 1 + λ/2 is the unique positive scalar such that: ( i ) RPCI  ∈  [0,1] for all admissible inputs; and (ii) the strict order RPCI(X) > RPCI(X ′ ) is preserved whenever  σ (X) <  σ (X ′ ) at equal  μ . Any divisor D ′  < D would allow RPCI > 1 for large PCI ₀  near 1. Any divisor D ′  > D would compress the range unnecessarily, reducing sensitivity near the boundaries. Among all linear normalisations satisfying ( i ) and (ii), D = 1 +  λ /2 achieves the widest admissible range, maximising discriminability between policy configurations. Remark A.4.2 (λ Sensitivity). The normalisation divisor D = 1 + λ/2 grows linearly in λ. For the Marikana calibration (λ = 0.10), D = 1.05 — a 5% compression. For λ = 0.50, D = 1.25 (25% compression). The sensitivity of RPCI to λ is analysed formally in Section 4.6 of the main text (Table 8), which confirms that RPCI remains in the Convergence Zone (PCI ≥ 0.70) across the full range λ  ∈  [0.00, 0.80] for the SCIPRA-optimised scenario. The parameter  λ  should be interpreted as a policy designer ’ s tolerance for cross-domain imbalance: λ = 0 implies indifference to dispersion; large λ imposes strong balance requirements. A.5 Theorem A.2: Strict Monotone Convergence (Proposition 2, Main Text) Notation: RPCI from Eq. (A.2); σ(X) = weighted standard deviation; μ = PCI₀ (held constant in this theorem); λ > 0 (strict inequality required). Theorem A.2 (Strict Monotone Convergence — Proposition 2, main text). Under Assumptions A.1–A.4 and for any λ > 0: RPCI is strictly decreasing in σ(X), holding μ = PCI₀ constant. Formally, for any X, X′  ∈  X with  σ (X) >  σ (X ′ ) and  ∑ᵢ   w ᵢ X ᵢ  =  ∑ᵢ   w ᵢ X ′ᵢ , it follows that RPCI(X) < RPCI(X ′ ). Proof. Part (a): Interior of feasible region (PCI₀ >  λσ (X)). When X lies outside the degeneracy region D₀ (Equation A.3), the  max( 0, ·) floor is not binding and Equation (A.2) reduces to: RPCI( σ)  =   (μ −  λσ ) / (1 + λ/2).    (A.11) With μ = PCI₀ held constant (Assumption A.4), RPCI is a function of σ alone. Taking the partial derivative (extending Equation 10 of the main text to the normalised case): ∂RPCI / ∂ σ  =   −λ / (1 + λ/2 )  <    0  for  all λ > 0.    (A.12) Since λ > 0 and D = (1 + λ/2) > 0, the partial derivative is strictly negative.  Therefore  RPCI is a strictly affine decreasing function of σ(X) in the interior of X. Part (b): Boundary of feasible region (PCI₀ ≤  λσ (X)). When X ∈ D₀, RPCI(X) = 0. For X′ with σ(X′) < σ(X) and equal μ, if PCI₀ >  λσ (X′) then RPCI(X′) > 0 = RPCI(X), and strict monotonicity holds. If both X and X′ ∈ D₀, then RPCI(X) = RPCI(X′) = 0; strict monotonicity does not hold at the boundary, but both configurations represent complete policy divergence and are observationally indistinguishable in  SCIPRA’s  governance interpretation. For the Marikana empirical scenarios (λ = 0.10, PCI₀ ∈ [0.456, 0.781]), the degeneracy condition PCI₀ ≤ 0.10 × σ is never satisfied (since 0.10 × 0.290 = 0.029 ≪ 0.456), so this case does not arise in practice. Part (c): Relationship to main text Equations (9) –( 10). Equations (9) –( 10) of the main text derive the result for the un-normalised  RPCI_raw  = PCI₀ −  λσ (X), obtaining ∂ RPCI_raw /∂σ = −λ. Equation (A.12) extends this to the normalised case, obtaining ∂RPCI/∂σ = −λ/(1+λ/2). The two are related by the scale factor 1/D. Both are strictly negative for λ > 0, confirming that normalisation preserves the monotone convergence property.  Remark A.5.1 (Optimisation Interpretation). RPCI may be interpreted as a regularised objective function of the form: RPCI = (mean performance) − λ × (dispersion penalty), normalised to [0,1]. This structure is analogous to mean-variance optimisation in portfolio theory (Markowitz, 1952), where λ plays the role of a risk-aversion coefficient. In SCIPRA, λ encodes a governance designer’s aversion to cross-domain policy imbalance: λ = 0 is risk-neutral (mean performance only); large λ is highly risk-averse (strong balance requirement). Theorem A.2 establishes that RPCI is the unique normalised linear  regulariser  that is both bounded and strictly monotone in dispersion for any λ > 0. A.7 Corollaries and Policy  Implications  ( Extending Propositions 1–2) These corollaries extend Theorem A.1 (Proposition 1) and Theorem A.2 (Proposition 2). Cross-references to main text are explicit. Corollary A.7.1 (Sectoral Optimum Penalty — extends Theorem A.2). A policy configuration that maximises one domain score at the expense of others achieves strictly lower RPCI than a configuration with the same PCI₀ but more balanced domain scores, provided λ > 0. Proof. Let X be a sectoral optimum with high σ(X), and X′ a balanced configuration with σ(X′) < σ(X) and equal μ = PCI₀. By Theorem A.2, RPCI(X) < RPCI(X′) Application to Marikana: Scenario A (X_I=0.82, X_S=0.12, σ=0.282) is the sectoral optimum; Scenario C (X_I=0.75, X_R=0.80, X_S=0.79, σ=0.022) is the balanced configuration. Despite Scenario A having a higher Investment score, Corollary A.7.1 predicts RPCI(A) < RPCI(C), confirmed numerically: 0.407 < 0.752. Corollary A.7.2 (Convergence Reward — extends Theorems A.1 and A.2). For fixed μ = PCI₀, RPCI is maximised when σ(X) = 0 (all domain scores equal to μ), yielding  RPCI_max  = μ / (1 + λ/2). This maximum is strictly less than PCI₀ for all λ > 0. Proof. If Xᵢ = μ for all  i , then (Xᵢ − μ) = 0 for all  i , so Var(X) = 0 and σ(X) = 0. By Equation (A.10): RPCI  =    max( 0, μ − 0) / (1 + λ/2 )  =   μ / (1 + λ/2 )  <    μ  =   PCI ₀  for  λ > 0.  Interpretation: Even at perfect cross-domain balance (σ = 0), RPCI is discounted below PCI₀ by a factor of 1/(1+λ/2). For λ = 0.10: discount = 1/1.05 ≈ 4.8%. This conservative discounting is by design — it ensures that no governance configuration receives a perfect RPCI = 1 unless λ = 0 and PCI₀ = 1, making the framework resistant to trivial perfect-score outcomes. Equation numbering continues from the main text (Equations 1-10) for coherence. B.1.1 TF -IDF Feature Weighting The term frequency-inverse document frequency weight for term  t  in document  d  from corpus  D  is defined as: tf- idf ( t, d, D )  =    tf ( t, d )  ×   log [ (1 + |D|) / (1 +  df ( t, D) ) ]    +  1 where  tf ( t, d)  =  log( 1 +  count( t, d)) (sublinear smoothing),  df ( t, D)  = number of documents containing term  t , and |D| = corpus size. The ‘+1’ additive smoothing prevents zero-division on unseen terms. This formulation (scikit-learn default with  sublinear_tf  = True) was chosen over raw term frequency for three reasons: ( i ) it reduces the influence of high-frequency non-informative terms; (ii) it produces interpretable feature weights that can be directly inspected alongside the domain lexicons; and (iii) it avoids the vocabulary size and training data requirements of contextual embedding models such as BERT, which would be disproportionate for a specialised 87-document corpus. The limitations of this choice relative to transformer-based models are acknowledged in Section B.5. B.1.2 Salience  Dimension Scoring Let  V_dim  denote the lexicon (valence term set) for dimension dim ∈ {P, L, U}, and let  V_all  = V_P  ∪  V_L  ∪  V_U  denote the union of all valence terms. The raw salience score for document  d  on dimension dim is: score_dim (d )  =   ∑ _{ t  ∈   V_dim }  ① ( t  ∈  tokens(d ))  /    ∑ _{ t  ∈   V_all }  ① ( t  ∈  tokens(d ))            ( B.1) where  ①( ·)  is the indicator function. The denominator counts all valence-bearing tokens across all three lexicons present in  d , ensuring scores are comparable across documents of varying length. The VADER compound sentiment adjustment applied to urgency is: U_adjusted (d )  =    clip(  score _U (d )  +   0.10 × compound(d ),  0,  1  )           ( B.2) where  compound(d)  ∈  [-1, 1]  is the VADER compound score (Hutto & Gilbert, 2014). The ceiling of ±0.10 was selected through sensitivity testing on a 20-document hold-out sample drawn from the corpus, comparing VADER-adjusted versus unadjusted urgency scores against annotator judgements. Adjustments exceeding 0.10 produced no improvement in annotator agreement and inflated urgency scores for documents with incidentally positive sentiment. B.1.3 Stakeholder  Acceptance Score ( S_score ) Let  G  = {government, community, investor, NGO, labour} be the set of stakeholder groups,  D_g   ⊆  D  the subset of documents attributed to group  g , and  pʲ (d)  the SVM-predicted probability of pro-integration stance for document  d . The group-level acceptance score is: acc( g)  =   (1 / | D_g |)  ×  ∑_{ d  ∈   D_ g }   p ʲ (d)           ( B.3) The  S_score  is then the SIC-weighted mean across stakeholder groups: S_ score   =   ∑ _{ g  ∈   G}   SIC _g   ×  acc( g)  /    ∑ _{ g  ∈   G}   SIC _g            ( B.4) where  SIC_g  is the Stakeholder Influence Coefficient for group  g  (Equation 2, main text; Marikana values:  SIC_community  = 0.749,  SIC_investor  = 0.770,  SIC_labour  = 0.807,  SIC_government  = 0.703, SIC_NGO = 0.686). The PCI is then computed per Equation (1) of the main text: PCI  =   w₁ ×  I_ score   +   w₂ ×  R_ score   +   w₃ ×  S_score            ( Eq. 1, main text) with w₁ = 0.30, w₂ = 0.35, w₃ = 0.35 for the Marikana case. The robustness-adjusted RPCI (Equations 5–8, main text) applies a variance penalty λ = 0.10 on the weighted standard deviation of domain scores. B.5.1 Scalability  to Larger Corpora For deployment at continental scale (e.g., multi-country SADC/ECOWAS monitoring), three adaptations are recommended. First, the SVM may be replaced with a linear kernel ( LinearSVC ) for corpora exceeding n = 5,000 documents, reducing training complexity to  O( n × V) while preserving comparable classification performance on high-dimensional TF-IDF features. Second, TF-IDF vectorisation supports incremental updates via  HashingVectorizer , enabling streaming document ingestion without full corpus refit. Third, for corpora requiring multilingual processing (isiZulu, Sesotho, French-language Francophone documents), language-specific tokenisation models and translated lexicons are required; this constitutes a priority direction for future development noted in Section 5.2 of the main text. B.5.2 Limitations  of the TF-IDF and SVM Approach The current implementation relies on TF-IDF bag-of-words features and a radial basis function SVM classifier. This constitutes an established but relatively standard machine learning pipeline rather than state-of-the-art natural language processing. Three specific limitations merit acknowledgement. First, TF-IDF does not capture word order, negation, or contextual polarity beyond the VADER adjustment; the phrase 'rights were denied' is treated equivalently to 'rights were upheld.' Second, the RBF-SVM is not well-suited to very large corpora due to quadratic training complexity. Third, domain-adapted transformer models (e.g., BERT fine-tuned on mining governance corpora) would likely improve salience classification accuracy, particularly for nuanced legitimacy claims. The term 'AI-enhanced' in the paper title is therefore intended to distinguish SCIPRA from purely expert-driven MCDM, not to claim transformer-level AI sophistication. Benchmarking against BERT-based classifiers is identified as a priority direction for future work (Section 5.2, main text). It should further be noted that the cross-validation metrics reported in Table 6 of the main text (accuracy = 81.6%, F1 = 0.837) are derived from the same 87-document Marikana corpus used to construct and illustrate the case argument. These metrics therefore reflect in-sample classification performance within a single case context rather than out-of-sample generalisation to independent governance settings. Cross-case validation — applying the trained pipeline to an independently annotated corpus from a second African mining jurisdiction (e.g., Zambia’s Copperbelt or Ghana’s Obuasi) — constitutes a priority direction for future work and is explicitly identified as such in Section 5.2 of the main text. C.9 Methodology  Transparency Audit Table C9.1 provides a structured audit of all 15 P, L, U scores, classifying each by subjectivity level, primary source, and reviewer verification pathway. A score is classified as ‘Low’ subjectivity when it is formula-derived or directly quantifiable from a primary source; ‘Low-Medium’ when a formula-derived base is adjusted by a single documented event whose magnitude involves calibrated judgement; and ‘Medium’ when the base score is set by theoretical anchoring with documented evidence support (as for Community Power). Audit summary:  Of the 15 P, L, U attribute scores, 8 (53%) are Low subjectivity (formula-driven or directly quantifiable), 6 (40%) are Low-Medium (formula-derived base with one calibrated adjustment), and 1 (7%) is Medium (Community Power, theoretically anchored with documented support). No score is classified as High subjectivity. The single  Medium  score (Community Power = 0.32) is the most theoretically sensitive; its ±0.05 sensitivity range [0.27, 0.37] does not materially alter the SIC ranking (community remains  third-highest , SIC ∈ [0.724, 0.774] vs. baseline 0.749). This is consistent with the Monte Carlo finding that  P( PCI ≥ 0.70) = 99.5% across 10,000 parameter combinations (Section 4.7, main text), confirming that the single Medium-subjectivity score does not threaten the framework’s conclusions. Validation against observed outcomes:  The P, L, U scores are externally validated by their consistency with the documented Marikana conflict dynamics. The stakeholder group with the largest Power–Legitimacy gap (Investor: 0.88 vs. 0.65, Δ=0.23) is empirically associated with the greatest governance failure ( Farlam  Commission attribution). The group with the lowest Power but highest  Legitimacy+Urgency  (Community: P=0.32, L=0.92, U=0.95) is the group whose exclusion is formally identified as the proximate cause of the conflict ( Farlam  Commission, 2015, p.487). This structural correspondence between the salience scores and the documented causal narrative provides external validity evidence independent of the SVM classification pipeline. Table C9.1.
